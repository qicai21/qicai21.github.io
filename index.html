初识linux系统文件

Published: Thu 07 January 2021
By Burgan

In Tips.

tags: linux ubuntu

经常性的搞不清楚linux下文件树,简单总结.

参阅: Ubuntu用户权限设置 史上最全Linux目录结构说明

不用碰,也碰不起的

  • boot drwxr-xr-x 系统
  • root drwx------ root专有
  • run drwxr-xr-x 系统自运行的服务
  • dev drwxr-xr-x 硬件配置,驱动文件

不用知道是什么,总之不用碰也碰不到的

  • opt drwxr-xr-x chrome, firefox这些大型文件通常在这里
  • proc dr-xr-xr-x
  • srv drwxr-xr-x
  • sys dr-xr-xr-x
  • var drwxr-xr-x

没啥看的,你都知道

  • home drwxr-xr-x 所有用户的集合,的确像个家.如同win下的users
  • tmp drwxrwxrwt 临时文件,会被自动清理, 比如浏览器缓存文件

两个引用过来的,相当于汇总

  • bin -> usr/bin lrwxrwxrwx 可直接执行, e.g.: pwd, ls
  • lib -> usr/lib lrwxrwxrwx 不可直接执行依赖库 e.g.: python3

重点看这里了

  • etc drwxr-xr-x 系统全局的配置文件
  • usr drwxr-xr-x 包含了命令库文件和在通常操作中不会修改的文件

usr下需要了解这些

usr不是user的缩写,而是"User System Resources",即面向用户使用的系统资源. 所以她包含了命令库文件和在通常操作中不会修改的文件,其地位类似Windows上面 的”Program Files”.

  • /usr/local 安装本地程序的一般默认路径
  • /usr/share 存放共享文件的目录
  • /usr/lib 就是python3, apt, dpkg这些了
  • /usr/bin 就是pwd, ls这些了
  • /usr/sbin 给超级用户root留的专属命令

配置文件放在哪?

  • ~/.config 执行程序的配置文件都放在这里.
  • ~/.*rc 各种程序的配置文件: e.g.: .bashrc, .vimrc

自己写的&下载的程序安装到哪?

比如neovim的插件最好各个用户能共享(虽然通常只有你一个用户,但理论上1即所有) 所以其恰当的位置是:~/.local/share, 事实上可以发现很多认识的app都装在这里.
