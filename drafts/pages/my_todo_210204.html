javascript on jupyter:

javascript运行环境,就是安装nodejs,并适配到jupyter上. 因npm有global和非global安装package之分,而ijavascript对zmq有依赖,所以node-gyp, zeromq, ijavascript 都要安装在一个目录下,即或是都为gloable或都不是.下边以-g模式为例:

sudo apt reinstall nodejs
sudo apt install build-essential
npm --version
npm list
npm -g list
npm install -g node-gyp  # 注意安装次序
npm install -g zeromq@6.0.0-beta.5
npm install -g ijavascript

ijsinstall  # 安装完ijavascript还要再与jupyter关联一下

Test using surface as a second display-screen

目前可行方案有:

Deskreen miraclecast Virtual Monitor Over VNC

Deskreen 这个是利用浏览器广播的,不能完全代替扩展屏,好在配置简单,使用方便. 但是好像对wifi网络有要求,家里的双频网2.4g就连不通,5g就能联通.

miraclecast 非常粗鄙的理解,这个相当于利用miracle服务,将ubuntu上 的NetworkManager.service & wpa_supplicant.service给嫁接了一下,做一个合并转发, 然后就能实现在局域网内的投屏.

VNC Server 与miracle类似,具体还没仔细看.

miraclecast不能正常工作 可能仅限于我的电脑.

关于安装miraclecast,反复测试安装后依旧不能正常工作,可能跟网卡的驱动有关, 也可能有其他原因,大致就是说miracle服务向wpa_supplicant发送命令时有长度限制, 15个字,但是部分系统或环境下因为wirless的名字的原因,长度超出了,就不能正常工作. 具体参阅: WARNING: supplicant: wpa_supplicant or driver does not support P2P

安装过程: 要求:

systemd版本>221,主要是关于kdbus和udev有要求,具体这些是什么我也不知道.

安装过程

  1. git clone to /usr/local/share/

  2. 确保已经满足了官方文档中的Requirements,特别是第4,5,6容易被忽视,缺什么就装什么

    Install GStreamer

  3. 安装必要的模块,之后编译autogen.sh

```bash cd /usr/local/share/miraclecast

sudo apt-get update sudo apt-get install ubuntu-dev-tools autoconf libtool libudev-dev sudo apt-get install libsystemd-dev glib-2.0 libperl-dev libgtk2.0-dev libreadline-dev

libtoolize; aclocal; autoconf

sudo ./autogen.sh

sudo ./configure sudo make sudo make install

```
